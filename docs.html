Documentation

Getting Started

Learn how to install, configure, compile, and integrate BhavyTr into your C projects.

Installation

Download the latest release from GitHub and extract it into your project directory. The package contains everything you need — headers, libraries, and the compiled binary.

bhavytr/
├── include/
├── lib/
├── bin/
└── BhavyTr.h

Setup & Compilation

Follow these steps to prepare your project and compile BhavyTr correctly on Windows with GCC.

1

Prepare Your Folder Structure

Your project root should look like this before compiling.

BhavyTr/
├── main.c
├── libbhavytr_shared.dll
├── include/
└── lib/
    └── gcc/
        ├── libbhavytr_shared.dll.a
        └── libbhavytr_static.a
2

Copy Files (Windows)

Run these commands in your terminal to set up the folder.

CMD 
mkdir BhavyTr
xcopy /E /I BhavyTr-v1.0.2\include BhavyTr\include
mkdir BhavyTr\lib\gcc
copy BhavyTr-v1.0.2\lib\gcc\libbhavytr_shared.dll.a BhavyTr\lib\gcc\
copy BhavyTr-v1.0.2\lib\gcc\libbhavytr_static.a BhavyTr\lib\gcc\
copy BhavyTr-v1.0.2\bin\gcc\libbhavytr_shared.dll BhavyTr\
3

Compile — Static (Recommended)

Static linking bundles everything into one executable — no DLL needed at runtime.

GCC 
gcc main.c -I include lib/gcc/libbhavytr_static.a -o app.exe
4

Compile — Dynamic

Use dynamic linking to keep the binary smaller. Requires the DLL at runtime.

GCC 
gcc main.c -Iinclude -Llib -lBhavyTr -o app.exe
5

Run Your App

Execute the compiled binary from your terminal.

CMD 
app.exe
⚠️ Make sure libbhavytr_shared.dll is in the same folder as your executable when using dynamic linking.

Folder Structure

Understanding the layout of BhavyTr helps you extend or debug it effectively.

  • include/ Public header files — the only headers you need to include in your code.
  • parser/ Input parsing layer — converts string expressions into tensor operations.
  • core/ The tensor engine — handles all computation and memory.

Basic Usage

Here's a complete example covering tensor creation, slicing, broadcasting, the expression engine, and cleanup.

C — main.c 
#include "BhavyTr.h"
#include <stdio.h>
#include <stdlib.h>

int main() {

    /* ── Create Tensors ── */
    Tensor* A = createTensorA("L[[1,2,3],[4,5,6]]");   // (2,3)
    Tensor* B = createTensorA("L[[10,20,30]]");         // (1,3)
    Tensor* C = createTensorA("L[[2],[3]]");            // (2,1)

    printTensor(A);
    printTensor(B);

    /* ── Slicing & Indexing ── */
    Tensor* s1 = sliceTensor(A, "I[1,2]");       // A[1,2]
    Tensor* s2 = sliceTensor(A, "S[:,1]");       // A[:,1]
    Tensor* s3 = sliceTensor(A, "S[0:2,1:3]");  // range
    Tensor* s4 = sliceTensor(A, "I[-1,-1]");    // negative index

    /* ── Broadcast Operations ── */
    Tensor* add1 = tensorAdd(A, B);
    Tensor* mul1 = tensorMul(A, C);

    /* ── Expression Engine ── */
    registerTensor('A', A);
    registerTensor('B', B);
    registerTensor('C', C);

    Tensor* e1 = ops("A+B");
    Tensor* e2 = ops("A*C");
    Tensor* e3 = ops("A-B");
    Tensor* e4 = ops("A/B");

    /* ── Memory Cleanup ── */
    freeTensorView(s1); freeTensorView(s2);
    freeTensorView(s3); freeTensorView(s4);
    freeTensor(add1);   freeTensor(mul1);
    freeTensor(e1);     freeTensor(e2);
    freeTensor(e3);     freeTensor(e4);
    freeTensor(A);      freeTensor(B);      freeTensor(C);

    return 0;
}

API Reference

A quick overview of the core functions available in BhavyTr.

📐 Tensor Creation

C 
Tensor* a = createTensorA("L[[1,2],[3,4]]");
Tensor* b = createTensor("[[5,6],[7,8]]");

➕ Operations

C 
Tensor* sum  = tensorAdd(a, b);
Tensor* prod = tensorMul(a, b);
Tensor* diff = ops("A-B");
printTensor(sum);

✂️ Slicing

C 
Tensor* elem  = sliceTensor(a, "I[0,1]");
Tensor* row   = sliceTensor(a, "S[0,:]");
Tensor* range = sliceTensor(a, "S[0:2,0:2]");

🧹 Memory Management

C 
freeTensor(a);       // Free an owned tensor
freeTensorView(s1);  // Free a slice / view tensor

Notes

  • Currently supports CPU execution only — no GPU required.
  • GPU support is planned for a future release.
  • Always call freeTensor or freeTensorView for every tensor you create to avoid memory leaks.
  • Use freeTensorView specifically for tensors returned by sliceTensor.
